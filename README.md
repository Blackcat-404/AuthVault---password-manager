#  Password Manager ‚Äî AuthVault 

**Your passwords. Your server. Your control.**

AuthVault is an open-source, self-hosted password manager designed for individuals who value privacy and want full control over their sensitive data. It allows you to host your own vault without relying on third-party cloud providers, with a focus on security and transparency.

## üì¶ Features

- **Self-hosted Freedom:** Host your vault on your own machine. No third-party servers, no cloud dependencies. Your data never leaves your hardware.  
- **100% Open Source:** Full transparency. Fork, audit, or contribute to the codebase on GitHub. No backdoors or hidden trackers.  
- **Zero-trust Architecture:** Encryption happens locally. Only you hold the keys to your data.  
- **Works Offline:** No internet? No problem. Access your passwords anytime, anywhere, independent of network availability.  
- **Built for Developers:** Clean **ASP.NET Core MVC** architecture with well-documented code. Perfect for learning, extending, or building custom features.  
- **True Ownership:** No subscriptions, no account limits, and no corporate surveillance. Deploy once, use forever.

## Tech Stack

- **Framework:** ASP.NET Core MVC 8.0  
- **Database:** SQL Server via Entity Framework Core 
- **Security:** AES-256-GCM local encryption for sensitive fields  
- **Frontend:** Razor Pages, Tailwind CSS

# Installation Instructions

### Prerequisites
- [.NET SDK 8.0+](https://dotnet.microsoft.com/download)  
- A modern web browser  
- **SQL Server** (recommended to run via docker)
- Email with 2FA security

---

### Installation & Local Run

1. **Clone the repository:**
   ```
   git clone https://github.com/Blackcat-404/AuthVault---password-manager.git
   cd AuthVault---password-manager/PasswordManager
   ```
2. **Pull the SQL Server image**

   SQL Server can be installed in multiple ways (Windows installer Azure, etc.), but we recommend using Docker for simplicity and consistency in development:
   ```
   docker pull mcr.microsoft.com/mssql/server:2025-latest
   docker images
   ```
   
3. **Database Setup (SQL Server via docker)**
   
   ```
   docker run -e "ACCEPT_EULA=Y" -e "SA_PASSWORD=CHANGE_ME" -p 1433:1433 --name sqlserver -d mcr.microsoft.com/mssql/server:2025-latest
   ```

   Then start your container:
   ```
   docker start sqlserver
   ```

   Update your `appsettings.json` connection string accordingly:
   ```json
   "ConnectionStrings": {
      "DefaultConnection": "Server=localhost,1433;Database=AuthVaultDb;User Id=sa;Password=CHANGE_ME;TrustServerCertificate=True"
   }
   ```

   Database update:
   ```
   dotnet ef database update
   ```

   Note: If you are having problems with the container, you have most likely entered a password that does not pass the `EULA`.

4. **Email Setup (for notifications, password recovery, etc.)**

   You can use any SMTP service (Gmail, Outlook, custom provider). Create an account and configure the settings in appsettings.json. Example with Google SMTP:
     ```json
     "EmailSettings": {
      "Host": "smtp.gmail.com",
      "Port": 587,
      "Username": "username@gmail.com",
      "Password": "your-app-password",
      "From": "AuthVault <from@gmail.com>"
     }
     ```

     > `"smtp.gmail.com"` - Default for Google, change it if you use another SMTP  
     > `587` - Default port for SMTP  
     > `"username@gmail.com"` - Email for sending messages  
     > `"your-app-password"` - Special 16-character password generated by your email provider for third-party applications.  
     > `"AuthVault <from@gmail.com>"` - Header of your messages

     **How to generate `app-password`?**
   
     **For Google SMTP:**
     - Google Accounts  
     - Search: `App passwords`  
     - Create your app password (name does not matter)  
     - Copy your `App password` and paste it into `appsettings.json` without spaces  
     
     Important: Replace all placeholders with your actual credentials. For Gmail, you may need an App Password for SMTP access.
    
   Example `appsettings.json` Template
     ```json
     {
        "Logging": {
          "LogLevel": {
            "Default": "Information",
            "Microsoft.AspNetCore": "Warning"
          }
        },
        "ConnectionStrings": {
          "DefaultConnection": "Server=localhost,1433;Database=AuthVaultDb;User Id=sa;Password=CHANGE_ME;TrustServerCertificate=True"
        },
        "EmailSettings": {
          "Host": "smtp.gmail.com",
          "Port": 587,
          "Username": "username@gmail.com",
          "Password": "your-app-password",
          "From": "AuthVault <from@gmail.com>"
        }
     }
     ```
  
     Open any text editor and change `appsettings.json` file.

6. **Enable HTTPS (Required for Email Confirmation)**

    Before running the project for the first time, generate and trust a local development certificate:
    ```
    dotnet dev-certs https --clean
    dotnet dev-certs https --trust
    ```
    **Important:** Close all browser pages, and there will be no problems with the https certificate.

    This ensures:
     - Secure cookies work correctly
     - Email confirmation links open without SSL errors
     - Authentication flows function properly    

7. **Run the application locally:**

   ```
   dotnet run
   ```

8. **Open your browser and navigate to `https://localhost:5001` (or the URL printed in the console).**
